# Prompt: Code Review

**Objetivo:** Realizar uma revisão de código minuciosa, focando em boas práticas, segurança, performance e manutenibilidade.

**Instruções:**
Atue como um Engenheiro de Software Sênior especialista em Go. Analise o código fornecido abaixo e forneça um feedback estruturado.

**Critérios de Análise:**
1.  **Corretude:** O código faz o que deveria fazer? Existem bugs óbvios?
2.  **Segurança:** Existem vulnerabilidades (ex: SQL Injection, XSS, vazamento de dados)?
3.  **Performance:** Existem gargalos de performance ou alocações de memória desnecessárias?
4.  **Legibilidade:** O código é fácil de entender? As variáveis e funções têm nomes descritivos?
5.  **Padrões:** O código segue as convenções do projeto e da linguagem (Idiomatic Go)?
6.  **Testes:** Existem testes cobrindo as mudanças? Eles são adequados?

**Formato da Saída:**
- **Resumo:** Uma breve descrição do que o código faz.
- **Pontos Fortes:** O que foi bem feito.
- **Sugestões de Melhoria:** Lista de melhorias com exemplos de código (diffs).
- **Problemas Críticos:** (Se houver) Bugs ou falhas de segurança que impedem o merge.
- **Nota:** (Opcional) Uma nota de 0 a 10 para a qualidade do código.
