# Prompt: Documentation

**Objetivo:** Gerar documentação clara e completa para o código.

**Instruções:**
Escreva a documentação para o código fornecido.

**Tipos de Documentação:**
1.  **Godoc:** Comentários acima de funções, structs e interfaces. Deve começar com o nome do elemento.
2.  **Exemplos:** Exemplos de uso prático (Executable Examples).
3.  **README:** Se for um pacote ou módulo, sugira seções para o README (Instalação, Uso, Features).

**Requisitos:**
- Use português claro e técnico.
- Explique *o que* faz e *como* usar.
- Mencione parâmetros, retornos e possíveis erros.
